# Architecture Notes

This file stores approved architecture decisions, feature decisions, bug design decisions, and refactoring scope decisions.

---

## How to Add an Entry

Paste the output from the relevant workflow step:
- **Feature decision** → architecture.md Step 6
- **Bug design decision** → bug-fixing.md Step 5 (when architecture change is required)
- **Refactoring scope** → refactoring.md Step 5

Use the format below. Add new entries at the bottom.

---

## Entry Format

```
### [Feature / Bug / Refactor Name]
**Date:** YYYY-MM-DD
**Type:** Feature | Bug Design | Refactor

**Decision:**
What was decided in one or two sentences.

**Options considered:**
- Option A: [why rejected]
- Option B: [why rejected]

**Behavioral contract:**
What inputs, outputs, and side effects are guaranteed by this decision.

**Status:** Approved | Cancelled
```
